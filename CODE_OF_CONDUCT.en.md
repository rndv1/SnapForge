# SnapForge Code of Conduct

**Language:** [Русский](CODE_OF_CONDUCT.md) | English

SnapForge is an open-source project for developers. We want issues, pull requests, reviews, and discussions to stay professional, useful, and safe for contributors with different levels of experience.

## Expected Behavior

- Communicate respectfully and stay on topic.
- Critique ideas, code, and decisions instead of people.
- Keep technical feedback specific: what is wrong, why it matters, and how it can be improved.
- Respect other people's time by checking existing issues and discussions before opening new ones.
- Help newcomers understand context when questions are asked in good faith.

## Unacceptable Behavior

- Insults, harassment, humiliation, or discrimination.
- Aggressive comments, threats, or pressure toward contributors.
- Publishing private information without permission.
- Spam, repeated noise, or intentional off-topic disruption.
- Any behavior that prevents constructive project work.

## Reporting a Problem

If you see a code of conduct issue, report it through a GitHub issue or contact the maintainer through their GitHub profile. If the report is security-related, do not publish details publicly and follow the [security policy](SECURITY.en.md).

Please include:

- a link to the discussion, issue, or pull request;
- a short description of what happened;
- why the behavior made project work harder or unsafe.

## Maintainer Response

Maintainers may remove comments, close discussions, ask a contributor to adjust their tone, or restrict participation when behavior repeatedly violates this code of conduct.

These rules are not decoration. They exist so the project can stay calm, useful, and professional.
