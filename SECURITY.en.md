# SnapForge Security Policy

**Language:** [Русский](SECURITY.md) | English

Thank you for helping keep SnapForge safe. If you find a vulnerability, please do not publish technical details in public issues, pull requests, or discussions.

## Supported Versions

| Version | Security Support |
| --- | --- |
| Latest release | supported |
| `main` | supported |
| Older releases | best effort only |

## Reporting a Vulnerability

The preferred path is GitHub private vulnerability reporting:

1. Open the repository **Security** tab.
2. Choose **Report a vulnerability** if it is available.
3. Describe the issue, expected impact, and minimal reproduction steps.

If private reporting is unavailable, open a public issue without technical details titled `Security contact request`. A maintainer will follow up through GitHub so the details can be discussed privately.

## What to Include

- affected version or commit;
- operating system and launch mode;
- short impact summary;
- minimal reproduction steps;
- any constraints that help verify the issue safely.

## Expected Response

- Acknowledgement: within 7 days.
- Initial triage: within 14 days.
- Fix and release: depends on complexity, but confirmed vulnerabilities will be prioritized.

## Scope

This policy covers SnapForge source code, release archives, CLI behavior, the local web interface, and artifact publishing. Bugs without security impact should be reported as regular bug reports.
